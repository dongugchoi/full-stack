<script>태그를 이용한 외부 API 사용

브라우저가 해당 스크립트를 불러와서 HTML파일에 포함한다음 실행
API에서 제공하는 함수나 객체들을 전역으로 생성할 수 있어 쉽게 사용할
수 있다.
다양한 기능이 내장되어 있어서 개발시간을 크게 줄여준다.

외부 스크립트를 우리가 제어하지못해서, 만약 스크립트가 악성 코드로
변질되면 보안문제가 생길 수 있다.
많은 JSSDK는 크기가 크고, 필요 없는 기능도 포함되어 있는 경우가 있어
페이지 로딩속도가 느려질 수 있다.

Software Development Kit는 개발자가 특정 기능이나 서비스에
쉽게 접근할 수 있도록 지원하는 도구의 모음

서비스가 제공하는 기능들을 간편하게 사용할 수 있도록 다양한 기능과
도구를 포함하는 모음

<script>로 요청했을 때 CORS가 발생하지 않는가? 
<script>를 html에 추가하고 나면 스크립트를 html의 일부로 취급을 한다.

axios또는 fetch를 이용한 직접 통신
- 브라우저에서 서버로 요청을 보낼 수 있는 HTTP 클라이언트 라이브러리다.

서버에 직접 요청을 해서 필요한 데이터만 가져오기 때문에 성능이 좋다.

데이터가 응답으로 올 때 그 데이터를 가공하거나, 응답 시 특정 조건에
따라 에러처리 로직을 쉽게 추가할 수 있다.

CORS 이슈가 발생할 수 있다.
